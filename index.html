The fulfilment flow, on Temporal

A parent RequestWorkflow spawns a child LinePlanWorkflow per line; task execution and rollback run as activities that call external app capabilities. In-flight state lives in workflow history — the state machines run inside the workflows. These are the real call sequences, traced from the executor/ code.

Entry 1 · Create

CreateFulfilment → start workflow → execute

The API validates, persists the request (idempotent on tenant + key), plans the lines, then starts the RequestWorkflow and returns immediately — execution runs durably in the background. The parent spawns a child per line; each child dispatches recipe steps as ExecuteTask activities. SYNC returns inline; ASYNC kicks off external work and parks; MANUAL dispatches nothing and parks — both await a task_callback signal.

Entry 2 · Report

An external callback resumes a parked task

A parked ASYNC or MANUAL task waits on the task_callback signal. The caller that completes it is outside the engine — a human clicking Complete in an ops surface, or another system posting its own result. ReportTaskResult runs the validation ladder, then signals the child LinePlanWorkflow directly, which applies the result and dispatches any unlocked successors.

Admin · Cancel / Compensate

Rollback as a Saga of workflows

compensate=false cancels the workflow — a clean forward stop. compensate=true (the default) signals the parent, which spawns a CompensationWorkflow per line; it walks that line's completed tasks in reverse order, invoking CompensateTask for each step that declares a compensation_key. The same machinery fires organically when a line fails under the ABORT failure policy.

Durable by history

Workflow history is the source of truth for in-flight state. A worker crash resumes mid-flight — no lease store, no recovery scanner.

Retries are free

ExecuteTask retries 5× with backoff; CompensateTask up to 10×. The workflow code stays a plain state machine.

Waits without polling

Async and manual steps park on the task_callback signal. An external human or system resumes them exactly when the work is done.
